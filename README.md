# 基于人工智能的服装成衣质量检测系统

## 项目背景

服装制造行业在成衣质检环节长期面临人工成本高、检测标准不一等痛点。本项目基于深度学习目标检测技术，构建了一套能够自动识别服装常见表面缺陷的智能检测系统，实现了从模型云端训练到移动端本地部署的完整链路。

## 核心工作与贡献说明

本项目的基础网络架构与原始成衣缺陷图像数据集由指导老师提供。在此基础上，本人的核心工作集中在数据工程、模型训练调优以及移动端算法修正三个方面：

1. **数据标注与预处理**
   * 基于老师提供的原始成衣图像，自主编写了数据标注辅助脚本。
   * 独立完成了常规矩形框与旋转框的数据集精准标注工作，为后续模型训练提供了可靠的数据基础。

2. **模型调优与云端训练**
   * 负责在云端算力平台搭建训练环境，完成了常规检测模型与 OBB 旋转框模型的训练。
   * 探索了在小样本数据集下的训练策略，通过增大批次大小与开启混合精度训练，有效提升了模型的收敛稳定性。

3. **移动端 Android 应用全栈开发**
   * 独立从零开发了用于实时检测的 Android 移动端应用程序。
   * 在移动端应用中创新性地集成了“布料缺陷检测”与“普通物体检测”的切换按钮，实现了多模型的灵活调用。
   * 在部署阶段，独立排查并修复了 OBB 模型输出通道解析错位导致的推理崩溃问题，确保了复杂模型在移动端的稳定运行。

## 模型性能指标

经过训练与验证，模型在各项缺陷类别上的平均精度表现如下：

| 缺陷类别 | 缺陷特征 | 使用模型 | 精确率 | 召回率 | mAP50 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **破洞** | 边界清晰度高 | 常规检测框 | 0.932 | 0.887 | 0.928 |
| **污渍** | 边界模糊、易与纹理混淆 | 常规检测框 | 0.505 | 0.412 | 0.444 |
| **线头** | 细长、多角度倾斜 | 旋转检测框 | 0.882 | 0.825 | 0.859 |

总体平均精度 mAP50 约为 0.465。其中，常规框对破洞的识别较为精准；OBB 模型有效改善了线头的检测效果；污渍的识别率因样本背景干扰较大，仍有提升空间。

## 核心代码结构

* `train.py` 与 `obb_train.py`：常规检测模型与旋转框模型的云端训练脚本。
* `annotate.py` 与 `annotate_obb.py`：本地轻量级图像标注辅助脚本。
* `android_app/`：本人独立开发的 Android 移动端应用源码，包含布料检测与普通物体检测双模式切换功能，核心推理逻辑位于 `FabricDetector.kt`。
* `colab_guide.md`：云端算力平台环境配置与训练指南。

## 改进方向

目前的系统在纯色衣物表面表现较好，但在带有复杂图案的服装上，背景干扰依然会导致误检。未来的改进方向主要集中在扩充复杂背景数据集，以及探索能够抑制背景特征干扰的优化算法。

## 智能质检桌面控制台 (GUI)

除了纯命令行调用，本项目还包含了一个基于 PySide6 + Fluent UI 的跨平台智能桌面控制台，供非研发人员使用。

### 核心特性
* 采用 Fluent Design 亚克力半透明与亮色主题设计，提供现代化的交互体验。
* 支持**服装色差等级评估**与**标签对齐与缺陷检测**双模式，一键式导入待测图片。
* 结果可视化展示：直观显示瑕疵区域，并输出详细的数值指标分析（如瑕疵面积、绝对色差分数、色差等级评定等）。

### 启动说明
1. 确保已正确配置依赖环境，详见项目依赖说明。
2. 运行打包编译后的 `app.exe`，或在命令行中执行源码启动：
   ```powershell
   python app.py
   ```
3. 在启动的图形界面中进行可视化操作。用于展示和测试的样本图可从 `color_tests/` 和 `label_tests/` 目录中选取。
